Case 1

Knight Capital Trading Disaster (2012)

In less than an hour, one deployment mistake escalated into a catastrophic business loss. The case shows why the full chain matters in real systems.

Error

Legacy trading code was not correctly removed when the new system was deployed.

Failure

The old code was accidentally activated and generated massive unintended trades.

Hazard

The system could execute large trades automatically without effective human control.

Risk

In a live high-frequency market, the loss became almost certain and extremely severe: $440M in 45 minutes.

Lessons Learned

  • Legacy code should be removed completely, not just ignored.
  • Deployment testing and release controls are essential.
  • Automated systems need safeguards such as limits and circuit breakers.
