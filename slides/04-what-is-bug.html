What is a Bug?

  • September 9, 1947: The first actual case of bug being found. It was a real moth stuck in a relay in the Harvard Mark II.
  • A bug is not just “bad code” — it is a mismatch between the expected result and the actual result.
  • If a test says Given conditions, When an action happens, and Then a result should be observed, a bug is the failure of that expectation.

Key idea: if the observed outcome is different from what the test promised, that difference is the bug.

Given

User enters correct username and password

→
When

User clicks Login

→
Then

System should open the dashboard

Expected

Dashboard loads successfully

Actual

Error message appears and login fails

Bug

The observed behaviour does not match the expected behaviour.
