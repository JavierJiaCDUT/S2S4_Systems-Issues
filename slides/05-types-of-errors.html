Three Types of System Errors

Different bugs appear at different moments in development: some are caught before the program runs, some crash during execution, and some quietly produce the wrong result.

</>

Syntax Error

Mistakes that break the grammar rules of the programming language, usually caught by the compiler or interpreter.

Detected: Before or during compilation
Typical signs: Red underline, compile error, parser stops
Example: Missing semicolon, unmatched bracket, misspelled keyword
!

Runtime Error

The code starts running, but the system crashes or throws an exception when it reaches a problematic situation.

Detected: While the program is executing
Typical signs: Crash, exception, timeout, memory problem
Example: Division by zero, invalid memory access, out of memory
≈

Logic Error

The program runs without crashing, but the reasoning or algorithm is wrong, so the output is incorrect.

Detected: During testing, review, or user feedback
Typical signs: Wrong result, wrong decision, hidden edge case
Example: Reversed sort order, wrong condition, missing boundary case
